React Hooks 的使用原理和最佳实践
中等 热度: 95% ▼
React Hooks 是 React 16.8 引入的新特性，允许在函数组件中使用状态和其他 React 特性。主要包括 useState、useEffect、useContext 等...
React Hooks 函数组件 状态管理
核心概念：
1. useState: 在函数组件中添加状态
2. useEffect: 处理副作用，相当于 componentDidMount、componentDidUpdate 和 componentWillUnmount 的组合
3. useContext: 消费 Context 值
4. useReducer: useState 的替代方案，适用于复杂状态逻辑

使用规则：
- 只在函数最顶层调用 Hook
- 只在 React 函数中调用 Hook
- 使用 ESLint 插件确保规则遵循

最佳实践：
- 使用多个 useState 分离不相关的状态
- useEffect 的依赖数组要包含所有使用的变量
- 自定义 Hook 提取组件逻辑

JavaScript 闭包 作用域 内存管理
定义：
闭包是指有权访问另一个函数作用域中变量的函数，创建闭包的常见方式是在一个函数内部创建另一个函数。

形成条件：
1. 函数嵌套
2. 内部函数引用外部函数的变量
3. 外部函数被调用

应用场景：
- 模块化开发（IIFE）
- 数据私有化
- 函数柯里化
- 回调函数
- 防抖和节流

注意事项：
- 可能导致内存泄漏
- 性能考虑
- 及时清理不需要的引用
HTTP 和 HTTPS 的区别及工作原理

HTTP HTTPS 网络协议 安全
主要区别：
1. 安全性：HTTPS 使用 SSL/TLS 加密，HTTP 明文传输
2. 端口：HTTP 使用 80 端口，HTTPS 使用 443 端口
3. 证书：HTTPS 需要 SSL 证书
4. 性能：HTTPS 有额外的加密解密开销

HTTPS 工作原理：
1. 客户端发起 HTTPS 请求
2. 服务器返回证书
3. 客户端验证证书
4. 协商加密算法
5. 建立安全连接
6. 加密数据传输

优势：
- 数据加密传输
- 身份验证
- 数据完整性
- SEO 友好
Vue.js 响应式原理详解
困难 热度: 85% ▼
Vue.js 的响应式系统是其核心特性之一，通过数据劫持和发布订阅模式实现数据变化的自动检测和视图更新...
Vue.js 响应式 数据劫持 观察者模式
Vue 2.x 响应式原理：
1. 数据劫持：使用 Object.defineProperty 劫持对象属性
2. 依赖收集：在 getter 中收集依赖
3. 派发更新：在 setter 中通知依赖更新
4. 异步更新：使用 nextTick 批量更新

Vue 3.x 响应式原理：
1. Proxy 代理：使用 Proxy 替代 Object.defineProperty
2. Ref 和 Reactive：提供更灵活的响应式 API
3. 更好的性能：减少不必要的依赖收集

核心概念：
- Observer：观察者，负责数据劫持
- Dep：依赖收集器
- Watcher：观察者，响应数据变化
- Scheduler：调度器，管理更新队列
算法：二分查找的实现和应用
中等 热度: 78% ▼
二分查找是一种高效的查找算法，适用于有序数组。时间复杂度为 O(log n)，空间复杂度为 O(1)...
算法 二分查找 时间复杂度 数组
算法思想：
在有序数组中，通过比较中间元素与目标值，每次排除一半的搜索空间。

实现步骤：
1. 设置左右边界 left = 0, right = arr.length - 1
2. 计算中间位置 mid = (left + right) / 2
3. 比较 arr[mid] 与目标值
4. 根据比较结果调整边界
5. 重复直到找到或边界相遇

应用场景：
- 有序数组查找
- 查找插入位置
- 查找边界值
- 峰值查找

注意事项：
- 数组必须有序
- 防止整数溢出
- 边界条件处理
