# GLM集成测试指南

## 功能验证清单

### 1. 基础配置检查

- [x] 腾讯云语音识别配置正常
- [x] GLM API配置正常（AI_API_KEY已设置）
- [x] 应用启动无错误

### 2. 语音识别功能测试

1. 点击"开始面试"按钮
2. 对着麦克风说话
3. 检查是否能正确识别语音并显示文本

### 3. GLM AI回复功能测试

1. 确保语音识别正常工作
2. 说完一句话后等待识别完成
3. 检查AI是否自动生成回复
4. 观察响应时间是否在合理范围内（<2秒）

### 4. 多轮对话测试

1. 进行连续的语音对话
2. 检查AI是否能记住上下文
3. 验证对话的连贯性

### 5. 状态指示器测试

- 语音识别状态显示
- GLM配置状态显示
- 响应时间显示

## 预期行为

### 正常流程

```
用户说话 → 语音识别 → 文本显示 → GLM处理 → AI回复显示
```

### 状态指示

- 🎤 语音识别状态：显示当前识别状态
- (AI已连接)：GLM服务正常
- 响应时间：显示AI处理耗时

### 错误处理

- 语音识别失败：显示错误提示
- GLM API调用失败：显示错误信息
- 网络问题：自动重试机制

## 性能指标

- 语音识别延迟：<1秒
- GLM响应时间：<500ms（使用AirX模型）
- 端到端延迟：<2秒

## 故障排除

### 常见问题

1. **语音识别不工作**
   - 检查麦克风权限
   - 验证腾讯云配置

2. **AI不回复**
   - 检查GLM API配置
   - 查看控制台错误信息

3. **响应太慢**
   - 检查网络连接
   - 验证API配置是否正确

### 调试信息

打开开发者工具查看控制台输出：

- 语音识别事件
- GLM API调用日志
- 错误信息
